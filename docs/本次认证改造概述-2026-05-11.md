# 本次认证改造概述（2026-05-11）

本文档简要说明本次认证与会话改造的范围和结果，便于研发、测试、产品快速同步。

## 改造范围

- 移动端登录与会话流程（`mobile`）
- 后端当前用户鉴权解析（`backend`）
- 登录页交互方式调整

## 关键改动

- 会话令牌统一为 **Firebase ID Token**
  - 移动端请求头改为实时使用 `FirebaseAuth.instance.currentUser?.getIdToken()`
  - 业务侧本地 token 存储不再作为主链路依赖
  - 退出登录改为 Firebase 直接 `signOut`
- 后端 `get_current_user` 支持 Firebase ID Token
  - 优先校验 Firebase ID Token
  - 根据 Firebase claims（`email` / `uid` / `name`）查找或创建本地 `users` 用户
  - 保留旧 JWT 的兼容回退逻辑（便于平滑过渡）
- Email/Password 增加“邮箱验证后才能登录”
  - 用户邮箱未验证时禁止进入应用
  - 自动发送/重发验证邮件并提示用户先完成验证
  - 注册成功后不再直接进入应用
- 登录页交互优化
  - 移除顶部“登录/注册”双 TAB 切换
  - 在主按钮下方增加链接式切换（“没有账号？去注册 / 已有账号？去登录”）
  - 登录与注册逻辑仍全部走 Firebase

## 用户可感知变化

- 邮箱密码用户必须先验证邮箱，再可登录进入应用。
- 已登录用户的会话恢复基于 Firebase 当前用户状态。
- 当后端返回 401 时，客户端会触发 Firebase 退出并回到登录页。

## 说明

- 当前仓库中仍保留部分旧 token 相关兼容代码（如 no-op / fallback），用于过渡。
- 如需进一步收敛代码复杂度，可在后续版本彻底移除旧业务 token 交换链路。
